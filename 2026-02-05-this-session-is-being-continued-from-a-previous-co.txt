     Refactoring Plan: User, UserGroup, and Auth Services

     Problem Summary

     Based on exploration of the codebase, the current authentication and
     user management architecture has the following issues:

     1. Monolithic AuthService: IAuthService.cs (618 lines) handles:
       - Authentication (Login, Register, RefreshToken, ConfirmEmail,
     SetPassword)
       - User CRUD operations (GetAllUsers, GetUserById, UpdateUser,
     DeleteUser)
       - UserGroup CRUD operations (CreateGroup, UpdateGroup, DeleteGroup,
     GetAllGroups, GetGroupById)
     2. Missing Components for User Service:
       - No IUserService interface
       - No UserService implementation
       - No UserController
       - No UserFilter or UserRead DTOs
       - UserRepository doesn't follow BaseEntityQuery/ProjectToRead pattern
     3. UserGroup Service Issues:
       - No dedicated IUserGroupService interface (methods in IAuthService)
       - No dedicated UserGroupService implementation (methods in 
     AuthService)
       - UserGroupRepository lacks BaseEntityQuery/ProjectToRead pattern
       - UserGroupController uses manual response formatting (not 
     ApiResponse helper)
       - No UserGroupFilter or UserGroupRead DTOs
     4. Inconsistent Patterns:
       - AuthController and UserGroupController don't use ApiResponse helper
       - Manual try-catch blocks instead of global error handling
       - No DataTablesProjectedRequest for filtering
